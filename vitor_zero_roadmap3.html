02 O que é cada coisa

Entendendo o arsenal
de uma vez por todas

IDEs
com IA
O que são Ambientes onde você escreve ou gera código. Cursor = VS Code com IA integrada. Replit = editor online. Lovable, Rocket, Emergent = você descreve em texto e eles constroem o app.
Use agora Cursor free
Como usar Cursor é seu IDE principal. Configure com Claude e OpenRouter. Lovable e Rocket são ótimos para prototipar rápido — você descreve o que quer e em minutos tem algo funcionando para mostrar a um cliente.
LLM Routers
& APIs
O que são Portais que dão acesso a múltiplos modelos de IA via API. OpenRouter = acessa Claude, GPT, Gemini, Kimi e mais 200 modelos com uma única key. Groq = mesmo conceito mas ultra-rápido e gratuito.
Free tier
Como usar Crie conta no OpenRouter, pegue uma API key gratuita e conecte no Cursor. Agora seu IDE tem acesso a todos os melhores modelos sem pagar por vários planos. Use Groq para tarefas rápidas — é de graça e muito veloz.
Local LLMs
(Ollama etc)
O que são Modelos de IA que rodam na sua máquina, sem internet, sem custo. Ollama e LM Studio são as interfaces. Você baixa um modelo como llama3.1 ou qwen2.5-coder e usa pelo terminal ou interface gráfica.
Para depois
Quando usar Ignore por enquanto. Local LLMs são para quando você já tem produto funcionando e quer reduzir custos de API ou rodar modelos privados. Antes disso, são distração. OpenRouter já te dá acesso ao que precisa.
Deploy
Platforms
O que são Serviços que publicam seu site ou app na internet. Vercel = padrão para Next.js, deploy em 30 segundos com um comando. Netlify = similar. Cloudflare Pages = mais rápido globalmente.
100% gratuito Use agora
Como usar Suba seu projeto no GitHub → conecte no Vercel → deploy automático. Sempre que você fizer git push, o Vercel publica sozinho. Você nunca mais vai ter que "subir via FTP" nada.
GitHub
& Portfólio
O que são GitHub é onde você guarda e versiona seu código. Portfólio não é um site bonito — é evidência de que você resolve problemas. São seus repositórios públicos com projetos reais.
Prioridade máxima
Como usar Crie sua conta, suba o Nexus Hub com README explicando o que é e quais tecnologias usou. Esse é seu projeto #1 de portfólio. Cada projeto no CS50x vira mais um repositório. Em 90 dias você tem 4-6 projetos reais.

05 Perguntas reais

As dúvidas que você tinha
respondidas de uma vez

O que são as IDEs como Lovable, Rocket, Emergent?
São "geradores de app por prompt" — você escreve o que quer em texto e eles geram o código React/Next.js completo. Não são substitutos para aprender a programar, mas são ferramentas poderosas para prototipar rápido ou mostrar uma ideia a um cliente em horas. Pense neles como o ChatGPT do desenvolvimento web — geram 70% do trabalho, mas você precisa entender o código para corrigir os 30% que faltam.
O que é o Hermes e para que serve?
Hermes é um agente de IA que age de forma autônoma — navega em sites, preenche formulários, executa sequências de tarefas sem você intervir. Diferente de Claude (que conversa), Hermes faz. É útil para automações repetitivas: "pesquisa os 10 principais fornecedores desse produto e me manda um relatório" — ele pesquisa sozinho. Mas precisa de configuração e faz mais sentido quando você já tem um fluxo de trabalho estabelecido para automatizar.
Preciso de certificados para conseguir trabalho?
Certificados são evidência secundária. O que emprega é o projeto. Um recrutador que vê "CS50x Harvard" no LinkedIn fica mais atento — mas só se tiver um projeto real do lado para confirmar. A ordem certa: projeto → certificado → LinkedIn → cliente. Nunca: certificado → certificado → certificado → certificado (que é onde a maioria trava).
Preciso de faculdade para ser dev?
Não. O mercado de desenvolvimento — especialmente com IA — avalia pelo que você construiu e entregou, não pelo diploma. CS50x de Harvard é gratuito, online e tem o mesmo valor de marketing que um semestre de faculdade particular. O que conta: GitHub com projetos, clientes satisfeitos e capacidade de resolver problemas reais.
Quanto tempo para conseguir o primeiro freela?
Com foco total — 60 a 90 dias. Semana 1: GitHub + LinkedIn. Semana 2–6: CS50x + mini-produto. Mês 2: perfil no Kwork com o produto pronto. Mês 3: primeira proposta aceita. O fator que determina tudo não é talento — é consistência de 1h por dia no que importa, não em explorar ferramentas novas.
